Texdoc - development version

Getting it

git clone git://github.com/mpg/texdoc.git (first time)
git pull (later)

If you don't have git installed, you can get the current 'master' branch from github as an archive in zip or tar.gz format.

Setting up to use it

The explanations below use TdDev to denote the path to the directory holding the development version of texdoc. Names in ALL_CAPS are kpse variables; you can get their values using kpsewhich --var-value VARIABLE.

The command - Unix

Just symlink TdDev/texdoc.tlu to ~/bin/texdoc (assumes ~/bin is in your PATH before the TeX Live binaries).

The command - Windows

Here things are a bit different, since there are generally no symlinks, so texdoc.bat is actually a script calling texdoc.tlu after locating it in the texmf trees. So, just put a copy of TdDev/texdoc.tlu in TEXMFHOME/scripts/texdoc (you may need to create this directory).

The configuration file - both platforms

Since the name or meaning of some options used in the default configuration file may change, you need to use the development version of texdoc.cnf too. Install a copy (or symlink) of it as TEXMFHOME/texdoc/texdoc-dist.cnf (notice the -dist part). It will completely override the configuration files in TEXMFDIST and TEXMFLOCAL.

If everything is installed correctly, running texdoc -f should yield something similar to the following.

texdoc version: 0.47
Configuration files are:
    absent /home/mpg/texmf/texdoc/texdoc-linux-latest.cnf
(*) active /home/mpg/texmf/texdoc/texdoc.cnf
    active /home/mpg/texmf/texdoc/texdoc-dist.cnf
    absent /usr/local/texlive/texmf-local/texdoc/texdoc-linux-latest.cnf
    absent /usr/local/texlive/texmf-local/texdoc/texdoc.cnf
    disabled /usr/local/texlive/2008/texmf/texdoc/texdoc.cnf
(*) This is the recommended configuration file for your personal preferences.
