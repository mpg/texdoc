Texdoc

Texdoc is a tool aimed at making TeX related documentation easier to consult. Its primary function is to locate relevant documentation for a given keyword (typically, a package name) on your hard disk, and open it in an appropriate viewer.

It is part of the TeX Live distribution.

Documentation

The documentation for the current stable version is available online in the following formats:

  • User manual: pdf only.
  • Unix man page: html and pdf versions.

Mailing list

Any questions, features report and bug requests (wait... maybe it's the other way round) are welcome on the dedicated mailing list. Please note that you don't need to subscribe to the list in order to post a question and receive the answers.

Development version

The latest version of texdoc is available via anonymous Subversion from svn://tug.org/texdoc/trunk. It should not be too broken, since I'm using it daily, but be warned that the documentation may be horribly out of sync.

The recommended way of running texdoc from the svn (on Unix) is to symlink trunk/texdoc.tlu to ~/bin/texdoc and trunk/texdoc.cnf to TEXMFHOME/texdoc/texdoc-dist.conf. This last step is needed since the name or meaning of some options used in the default configuration file sometimes change.

If everything is installed correctly, running texdoc -f should yield something similar to the following.

texdoc version: 0.4+ svn r33
Configuration files are:
    absent /home/mpg/texmf/texdoc/texdoc-linux-latest.cnf
(*) active /home/mpg/texmf/texdoc/texdoc.cnf
    active /home/mpg/texmf/texdoc/texdoc-dist.cnf
    absent /usr/local/texlive/texmf-local/texdoc/texdoc-linux-latest.cnf
    absent /usr/local/texlive/texmf-local/texdoc/texdoc.cnf
    disabled /usr/local/texlive/2008/texmf/texdoc/texdoc.cnf
(*) This is the recommended configuration file for your personal preferences.

Known problems

  • Problems have been reported when trying to open pdf files with Adobe Reader 5.0.5 under windows when AR is already running.
  • Using TeX Live 2008 (DVD release), there was problems with recent versions of Adobe Reader on windows. They have been fixed in the tlnet update of 2008/09/12, by some changes to the TL wrapper scripts. Please upgrade bin-texdoc using tlmgr if you run into this problem.
